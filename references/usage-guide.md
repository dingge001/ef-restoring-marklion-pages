# 标记狮还原技能使用向导（稳定版）

适用场景：你希望按**两阶段固定流程**完成标记狮页面高一致还原，并降低跑偏与返工。

## 快捷命令
- `/EF-标记狮还原`
- 技能名称：`ef-restoring-marklion-pages`

## 一句话指令（推荐直接复制）

### 标准稳定流程（最推荐）
```text
/EF-标记狮还原 按两阶段固定流程执行：阶段一先抽离公共层（顶部/背景/侧栏）并产出共享模块，阶段二再做单页面还原；禁止 preview 整图贴图交付，局部问题必须局部修复。
```

### 默认像素级（新增）
```text
/EF-标记狮还原 默认按 1:1 像素级验收执行；若我未明确同意降级，不要用结构占位/大致相似交付。
```

### 像素级 + 响应式（新增）
```text
/EF-标记狮还原 默认按 1:1 像素级 + 响应式可用交付：保持 1920 基准对齐，同时提供 1366 等分辨率可用适配。
```

### 像素级 + 非全屏铺满（新增）
```text
/EF-标记狮还原 除 1920 基准外，非 F11 宽屏窗口（如 1536x864）也必须可用且无左右大留白；默认采用按宽度铺满并顶部对齐策略。
```

### 顶部导航公共化（新增）
```text
/EF-标记狮还原 顶部导航菜单必须抽成公共组件并被所有页面复用，不能在每个页面各写一套热区和跳转。
```

### 阶段一：公共抽离
```text
/EF-标记狮还原 先执行阶段一公共抽离：确认并行路由、识别共享层、输出 shared-data + shared-layout + shared-layout.vue；阶段一未完成前不要进入单页开发。
```

### 阶段二：单页还原
```text
/EF-标记狮还原 在已完成共享层基础上还原这个页面：先做 data.js 校验清洗，再做切片/文本/交互分层渲染，最后做关键区域对齐验收。
```

### 强门禁（防再犯）
```text
/EF-标记狮还原 执行硬门禁：禁止 preview.png 冒充还原；禁止为修局部问题改全局 white-space/line-height；必须给出路由-画面-资源一致性验证结果。
```

## 推荐执行顺序
1. 确认改造边界（并行新增/替换旧页）+ 确认验收等级（默认像素级）+ 确认响应式策略（等比缩放/断点重排）。
2. 执行技术栈检测并绑定模板。
3. 完成阶段一共享层抽离并产出共享文件。
4. 进入阶段二单页还原并复用共享层。
5. 通过 build/dev、关键区域对齐与多分辨率验收。

## 常见问题与处理
- 页面“看起来像但不可维护”：通常是整图贴底。处理：撤销贴图，回到图层渲染。
- 页面“能跑但不像设计稿”：通常是先做了结构骨架没做图层坐标渲染。处理：回到 data.js，按切片/文本坐标做像素级收口。
- 页面“1920 准但 1366 崩”：通常是缺少响应式策略。处理：补等比缩放容器或断点重排，并做分辨率回归。
- 页面“非全屏左右有空白”：通常是桌面端始终 `scale(min(xScale,yScale))` 导致。处理：在宽屏窗口切到 `scale(xScale)` + 顶部对齐。
- 顶部菜单“有的页面能点有的不能点”：通常是热区分散写在各页导致漏改。处理：抽离公共顶部导航组件并全页面引用。
- 修了两处文字其他全乱：通常是改了全局文本策略。处理：回退全局改动，改为节点白名单。
- 路由切了画面没变：通常是数据源或资源目录仍指向旧页。处理：校验“路由-画面-资源”三点一致。
